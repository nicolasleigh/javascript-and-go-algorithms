## 🎯 题目回顾

给定火车进站顺序，比如：

```
输入：
3
1 2 3
```

要求输出所有 **可能的出站顺序**，但要满足 **栈的先进后出 (LIFO)** 规则。

---

## ✅ 思路回顾（DFS）

这个方法用 DFS 模拟火车的进出站过程，使用三个变量：

* `waitIn`：还未进站的火车队列（比如 `["1", "2", "3"]`）
* `stack`：当前在站内的火车（栈结构）
* `output`：已经出站的火车（结果）

---

## 🧪 举例说明：输入为 `1 2 3`

我们一步步分析 DFS 是怎么枚举出所有合法出栈顺序的：

### 初始状态：

```
waitIn: ["1", "2", "3"]

### 第一步：让 1 进栈

```
waitIn: ["2", "3"]
stack: ["1"]
output: []
```

#### 然后让 2 进栈：

```
waitIn: ["3"]
stack: ["1", "2"]
output: []
```

##### 然后让 3 进栈：

```
waitIn: []
stack: ["1", "2", "3"]
output: []
```

此时，waitIn 已空，不能进栈，只能出栈：

* 出 3 → stack: \["1", "2"], output: \["3"]
* 出 2 → stack: \["1"], output: \["3", "2"]
* 出 1 → stack: \[], output: \["3", "2", "1"]

得到第一个答案：`3 2 1`

---

### 回溯：在 `["1", "2"]` 时选择先出 2 而不是进 3

* 出 2 → stack: \["1"], output: \["2"]
* 进 3 → stack: \["1", "3"]
* 出 3 → stack: \["1"], output: \["2", "3"]
* 出 1 → stack: \[], output: \["2", "3", "1"]

得到：`2 3 1`

---

### 继续尝试更多路径...

每次都根据：

* **还能进站** → 模拟进栈
* **栈里有火车** → 模拟出栈

每条路径走到 `waitIn=[]` 且 `stack=[]` 就记录结果。

## 🧾 最终输出（字典序）

DFS 全部路径遍历完后，输出如下结果（按字典序排序）：

3 2 1
```

## 🧠 总结

这个方法的精髓是：

* 枚举所有合法进出栈序列（用 DFS）
* 每步都做两件事：

  1. 如果还能进栈，就试试进栈
  2. 如果栈不空，就试试出栈
* 只在所有火车都出站后记录结果
